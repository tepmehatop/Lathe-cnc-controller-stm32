# Lathe CNC Controller — STM32F4DISCOVERY

Контроллер электронного ходового винта (ELS) для токарного станка на базе STM32F407VGT6.
Миграция с Arduino Mega 2560: устранена задержка отображения 500 мс (SoftwareSerial) — STM32 обеспечивает <5 мс за счёт аппаратных UART + DMA.

## Описание

ELS синхронизирует продольную (Y) и поперечную (X) подачи с вращением шпинделя для нарезания резьбы, выполнения синхронной и асинхронной подачи, конусного точения. Основой является TIM1 (HardwareTimer) для генерации шагов и TIM5 (quadrature encoder) для отслеживания шпинделя.

### Почему STM32 вместо Arduino Mega

| Параметр | Arduino Mega 2560 | STM32F407VGT6 |
|---|---|---|
| Тактовая частота | 16 MHz | 168 MHz (Cortex-M4F + FPU) |
| UART (аппаратные) | 4 | 6 + DMA |
| Задержка отображения | ~500 мс (SoftwareSerial) | <5 мс (UART8 + DMA) |
| RAM | 8 KB | 192 KB |
| Flash | 256 KB | 1 MB |
| Таймеры с энкодером | ограничено | TIM1/TIM2/TIM3/TIM4/TIM5/TIM8 |

## Аппаратная часть

**Плата**: STM32F4DISCOVERY (STM32F407VGT6)  
**IDE**: PlatformIO + STM32duino (Arduino framework)  
**Программатор**: встроенный ST-Link/V2 (SWD), он же Debug UART (VCP)

### Подключённые устройства

| Устройство | Интерфейс | Параметры |
|---|---|---|
| DRO HS800-2 | USART2 RX=PA3 | 57600/8N1, пакеты 29 байт, CRC16/MODBUS |
| ESP32-S3 Display | UART8 TX=PE0 RX=PE1 | 115200/8N1, протокол `<CMD:value>` |
| Debug / CLI | USART1 PA9/PA10 (ST-Link VCP) | 115200/8N1 |
| LCD2004 | I2C1 PB6=SCL PB7=SDA | addr 0x27 (PCF8574), 400 kHz |
| Энкодер шпинделя | TIM5 CH1=PA0 CH2=PA1 | quadrature ×4, 1800 линий → 7200 ppr |
| Ручной энкодер | EXTI PC2=ChA PC3=ChB | 100 линий/об |
| Шаговик Y (продольный) | TIM1_CH1 STEP=PE9 DIR=PE10 EN=PE11 | 400 шагов/мм |
| Шаговик X (поперечный) | TIM1_CH3 STEP=PE13 DIR=PE14 EN=PE15 | 640 шагов/мм |
| Beeper | TIM4_CH1 PD12 | PWM |
| Тахометр | PE8 | цифровой выход |

### Масштаб моторов

- **Y**: 200 шагов/об × 4 микрошага / 2 мм шаг винта = **400 шагов/мм**
- **X**: **640 шагов/мм**
- Диапазон оборотов: 10–3000 RPM
- Максимальная частота шагов: 50 kHz

## Быстрый старт

### Зависимости

```ini
; platformio.ini
[env:stm32f407_discovery]
platform = ststm32
board = disco_f407vg
framework = arduino
```

### Сборка и прошивка

```bash
# Сборка
pio run -e stm32f407_discovery

# Прошивка через ST-Link
pio run -e stm32f407_discovery -t upload

# Мониторинг serial (115200)
pio device monitor -b 115200
```

## Режимы ELS

| Режим | Название | Описание |
|---|---|---|
| `FEED` | Синхронная подача | мм/об — подача синхронизирована со шпинделем |
| `AFEED` | Асинхронная подача | мм/мин — подача независима от шпинделя |
| `THREAD` | Нарезание резьбы | шаг резьбы в мм/об, синхронизация по меткам |
| `CONE` | Конусное точение | X/Y соотношение для конуса |

## Серийные команды

Подключение: **USART1 / ST-Link VCP, 115200 бод**.

| Команда | Описание |
|---|---|
| `HELP` | Список команд |
| `INFO` | Текущее состояние (режим, подача, позиция) |
| `START` | Запуск ELS |
| `STOP` | Остановка ELS |
| `MODE:N` | Установить режим (0=FEED, 1=AFEED, 2=THREAD, 3=CONE) |
| `FEED:N` | Установить синхронную подачу, мм/об (например `FEED:0.2`) |
| `AFEED:N` | Установить асинхронную подачу, мм/мин |
| `PITCH:N` | Установить шаг резьбы, мм |
| `STARTS:N` | Количество заходов резьбы |
| `ZERO:Y` | Обнулить позицию Y |
| `ZERO:X` | Обнулить позицию X |
| `LIMIT:ON` | Включить концевые выключатели |
| `LIMIT:OFF` | Отключить концевые выключатели |
| `55` | Загрузить тестовый пресет |

Пример сессии:

```
> INFO
Mode: FEED, Feed: 0.10 mm/rev, RPM: 450, Y: 0.000 mm, X: 0.000 mm

> MODE:2
> PITCH:1.5
> STARTS:1
> START
```

## Структура проекта

```
Lathe-cnc-controller-stm32/
├── src/
│   └── main.cpp               # setup() / loop()
├── Core/
│   ├── els_config.h           # Все пины, константы, параметры
│   ├── els_state.h / .cpp     # Глобальная структура состояния
│   ├── els_main.h / .cpp      # Инициализация и главный цикл (оркестрация)
│   ├── els_control.h / .cpp   # Замкнутый контур управления мотором (Etap 8)
│   └── els_serial.h / .cpp    # Обработка команд по UART (Etap 9)
├── Drivers/
│   ├── drv_dro.h / .cpp       # Парсер протокола HS800-2 (29 байт, CRC16/MODBUS)
│   ├── drv_display.h / .cpp   # Протокол ESP32-S3 дисплея (<CMD:value>)
│   ├── drv_lcd2004.h / .cpp   # HD44780 4-bit через PCF8574 по I2C
│   ├── drv_encoder.h / .cpp   # TIM5 (шпиндель) + EXTI (ручной энкодер)
│   ├── drv_inputs.h / .cpp    # GPIO входы с дребезгом 12 мс
│   ├── drv_stepper.h / .cpp   # TIM1 HardwareTimer PWM генерация шагов
│   └── drv_beeper.h / .cpp    # TIM4 PWM зуммер
├── Simulator/
│   └── ...                    # Симулятор DRO HS800-2 (Arduino sketch)
├── docs/
│   ├── pinout.html            # Интерактивная распиновка + визуальная схема
│   ├── schematic.html         # Автономная схема подключения
│   └── shield_recommendations.md  # Рекомендации по разработке PCB
└── platformio.ini
```

### Ключевые заголовочные файлы

**`Core/els_config.h`** — единственный источник всех аппаратных констант. При замене платы или изменении разводки достаточно отредактировать только этот файл.

**`Core/els_state.h`** — глобальная структура `ELSState`, содержит текущий режим, подачу, позиции осей, счётчики энкодера, флаги.

## Тестирование

### 1. Проверка Debug UART

```bash
pio device monitor -b 115200
# Ожидаем приветствие и ответ на команду INFO
```

### 2. Симулятор DRO

Скетч `Simulator/` загружается на Arduino/ESP32 и генерирует пакеты HS800-2 на TX.
Подключить TX симулятора → PA3 (USART2 RX) платы STM32.

```
> INFO
DRO: X=12.500 Y=0.000 Z=0.000  (пример — пакеты принимаются и парсятся)
```

### 3. Проверка концевиков

```
> LIMIT:ON
# Замкнуть PF0 на GND — должен загореться LED PF1, движение по Y заблокировано
```

### 4. Тестовый пресет

```
> 55
# Загружает предустановленные параметры для быстрой проверки всех режимов
```

### 5. Проверка шаговых моторов (без нагрузки)

```
> MODE:1
> AFEED:10
> START
# Y должен двигаться со скоростью 10 мм/мин
> STOP
```

### 6. Проверка резьбы

Подключить шпиндельный энкодер (PA0/PA1), вращать шпиндель вручную:

```
> MODE:2
> PITCH:1.0
> START
# Мотор Y должен синхронно следовать за шпинделем
```

## Параметры компиляции

Все аппаратные параметры сосредоточены в `Core/els_config.h`. Важные константы:

```cpp
// Масштаб осей
#define MOTOR_Y_STEPS_PER_MM   400.0f   // 200 * 4 / 2
#define MOTOR_X_STEPS_PER_MM   640.0f

// Энкодер шпинделя
#define SPINDLE_ENCODER_PPR    7200     // 1800 линий × 4

// Диапазон оборотов
#define RPM_MIN                10
#define RPM_MAX                3000

// Максимальная частота шагов
#define STEP_FREQ_MAX_HZ       50000

// LCD I2C
#define LCD_I2C_ADDR           0x27
```

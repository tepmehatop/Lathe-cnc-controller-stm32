## [BACKLOG-001] GCode: Токарный станок как ЧПУ через WiFi

**Приоритет:** Высокий  
**Компонент:** ESP32-S3 (JC4827W543) + STM32F407

### Описание

Реализовать работу токарного станка с GCode-программами по аналогии с 3D-принтером.
Пользователь с компьютера в локальной сети заходит на веб-страницу ESP32, управляет файлами GCode и запускает резку.

### Ссылки для изучения

- Пример реализации (WiFi + веб-интерфейс + GCode): https://github.com/kachurovskiy/nanoels/blob/main/h5/h5.ino
- Генератор GCode для токарного станка: https://github.com/kachurovskiy/lathecode

### Что нужно изучить в nanoels/h5.ino

1. Подключение к WiFi (STA режим, точка доступа или клиент)
2. Веб-сервер внутри ESP32 — локальная страница в браузере
3. Файловая система (SPIFFS/LittleFS) для хранения GCode файлов
4. Парсер GCode (G0/G1 линейное перемещение, G2/G3 дуга, M-команды)
5. Как ESP32 передаёт команды движения в основной контроллер

### Требования к веб-интерфейсу (ESP32 HTTP-сервер)

- **Список файлов** — отображение всех загруженных `.gcode` файлов
- **Загрузка** — upload файла с компьютера (multipart POST)
- **Просмотр** — отображение содержимого GCode в браузере
- **Редактирование** — встроенный текстовый редактор GCode
- **Визуализация** — 2D-траектория резца (SVG или Canvas)
- **Удаление** — удаление файла
- **Запуск** — старт выполнения GCode программы
- **Стоп/Пауза** — аварийная остановка

### Требования к выполнению GCode (ESP32 → STM32)

- ESP32 парсит GCode построчно
- Каждая строка преобразуется в команду перемещения (ось X/Y, скорость)
- Команда передаётся в STM32 по UART (USART3, протокол `<CMD:value>`)
- STM32 выполняет перемещение через шаговые двигатели
- ESP32 ждёт подтверждения (`<OK>`) перед следующей строкой
- Учёт нуля (G54 Work Coordinate System) — пользователь задаёт нулевую точку

### Поддерживаемые GCode команды (минимум)

| Команда | Описание |
|---------|----------|
| G0 Xn Zn | Быстрое перемещение |
| G1 Xn Zn Fn | Рабочая подача |
| G4 Pn | Пауза (мс) |
| G28 | Переход в домашнюю позицию |
| M0 | Программная остановка |
| M2/M30 | Конец программы |

### Расширение протокола ESP32 ↔ STM32

Новые команды для STM32 (дополнение к существующему протоколу):

```
<GCODE:G1 X10.5 Z-25.0 F100>   — выполнить строку GCode
<GCODE_STOP>                    — аварийная остановка
<GCODE_PAUSE>                   — пауза
<GCODE_RESUME>                  — продолжить
```

Новые ответы STM32 → ESP32:
```
<OK>           — строка выполнена, готов к следующей
<ERR:reason>   — ошибка выполнения
<POS:x,z>      — текущая позиция (периодически)
```

### Этапы реализации

1. **ESP32:** Изучить h5.ino — WiFi + AsyncWebServer + SPIFFS
2. **ESP32:** Файловый менеджер GCode (upload/list/delete/view)
3. **ESP32:** Парсер GCode (G0, G1, G4, M0, M2, M30)
4. **ESP32:** Веб-визуализатор траектории (Canvas 2D)
5. **STM32:** Обработка команды `<GCODE:...>` в drv_display.cpp
6. **STM32:** Прямое управление степперами из GCode (минуя ELS логику)
7. **Интеграция:** Handshake OK/ERR между ESP32 и STM32
8. **Тест:** Простая программа (проточка цилиндра) от начала до конца
